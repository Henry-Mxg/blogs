    大文件分片上传

    5/13/2023 前端随笔

    前言

    大文件上传以及超大文件上传都需要我们前端进行分片上传以达到优化性能，加快速度，增加用户体验的功能，大多数情况中，上传分为直接上传（小文件，如小于8M）和分片上传（大文件，大于8M），直接上传就不用多说了，直接上传就可以了，有必要的是我们可以提一下在直接上传时，前端获取当前文件的MD5值

    文中所有的 this _this 都是在data中定义的,文中实现了自定义进度条的显示,以及断点续传和秒传的功能,还有中断上传的功能

    获取MD5值

    1.首先，引入插件SparkMD5，import SparkMD5 from 'spark-md5'

    2.在onChange中计算文件的MD5值

    分片上传

    分片上传，其中的逻辑是：首先，需要把文件分成每片多少大小（比如每个分片大小是8M），总共分成了多少片，当前正在上传切片的索引是多少？知道这些就好搞了

    const chunkSize = 8 * 1024 * 1024; // 每个分片为8MB
const totalChunks = Math.ceil(file.size / chunkSize); // 表示文件被切割成的总分片数,totalChunks 的值决定了分片上传的循环次数，确保所有分片都能被上传

    其次：先获取到已经上传的切片信息，目的就是为了断点续传

    然后就是进行分片上传,分片上传中我们需要注意的问题1.分片上传失败,需要重新上传当前的分片,2.分片上传中,上传进度条应与上传一致,3.上传到一半退出上传,应中断文件上传,4.下次上传同样文件时应过滤掉已经上传过的文件,以下代码包含这些所有问题,并注有详细的解释

function uploadChunk(chunk,index,totalChunks,md5) {
        const formData = new FormData()

        formData.append('key', _this.MultipartUploadKey)
        formData.append('bucket', 'upload')
        formData.append('useJson', 'true')
        formData.append('uploadId', _this.uploadId)
        formData.append('partNumber', index)
        formData.append('contentMd5', md5)
        return  getUploadPartAPI(formData).then((res) => {

      }


async function uploadChunksInBatches(file,batchSize,resumeIndex = 0) {
        _this.isUploading = true;
        let start = resumeIndex * chunkSize
        _this.totalChunks = totalChunks; // totalChunks在data中定义
        // 循环每个批次
        for (let i = resumeIndex; i< totalChunks;i+=batchSize) {
          const end = Math.min(i+batchSize,totalChunks)
          const batchChunks = []
          const batchPromises = []
          // 重置进度计数器
          let batchUploadedChunks = 0;
          // 构建当前批次的切片数组
          for (let j = i;j<end;j++) {
            // 如果标志位为 true，中断上传，isAbort在data中定义
            if (_this.isAborted) {
              _this.$message.warning('上传中断')
              return;
            }
            const progress = (((i + 1) / totalChunks) * 100).toFixed(1);
            _this.updateProgressBar(progress);
            const chunk = file.slice(start,start+chunkSize)
            batchChunks.push(chunk)
            start+=chunkSize
            // 更新已上传切片数
            batchUploadedChunks++
            // 更新整体上传进度
            _this.uploadedChunks++;
            _this.updateProgressBar();
            // 计算切片得MD5值
            const md5Promise = new Promise((resolve, reject) => {
              const reader = new FileReader();
              reader.onload = () => {
                const spark = new SparkMD5.ArrayBuffer();
                spark.append(reader.result);
                const md5 = spark.end();
                resolve(md5);
              };
              reader.onerror = reject;
              reader.readAsArrayBuffer(chunk);
            });
            batchPromises.push(md5Promise) // 把每个切片得MD5值保存起来
          }
          const md5Values = await Promise.all(batchPromises);
          // 过滤掉已上传的切片
          // _this.getUploadedChunks()
          const uploadChunks = batchChunks.filter((chunk, index) => {
            const chunkIndex = i + index;
            const isUploaded = _this.part.includes(chunkIndex);
            return !isUploaded;
          });
          // 并发上传切片，返回上传的 Promise 对象
          if(uploadChunks.length > 0) {
            const uploadPromises = uploadChunks.map((chunk, index) =>
              uploadChunk(chunk, i + index, totalChunks, md5Values[index], i > resumeIndex) // 传递 MD5 值
            );
            try {
              // 等待所有上传请求完成
              await Promise.all(uploadPromises)
            }catch (error) {
              // 重试失败的切片上传
              const failedUploads = []
              for (let j = 0;j<uploadPromises.length;j++) {
                try {
                  await uploadPromises[j]
                }catch (error) {
                  // 记录上传失败的切片
                  failedUploads.push(j)
                }
              }
              // 对于上传失败的切片，进行重试
              for (const failedIndex of failedUploads) {
                const retryChunk = batchChunks[failedIndex]
                const retryMd5 = md5Values[failedIndex]
                await uploadChunk(retryChunk,i+failedIndex,totalChunks,retryMd5)
              }
            }
          }
        }
      }

    上述代码中，有个bug，就是当文件上传完成以后，你又点击了别的文件进行上传，进度条就不是从0开始了而是从上次的100%开始以此类推，所以我们在beforeUpload最上面this.uploadedChunks= 0，每次上传的时候，都把uploadedChunks置0即可
